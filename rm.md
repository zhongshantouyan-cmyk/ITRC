## 系統架構總覽

| 組件 | 技術 | 部署平台 |
|------|------|----------|
| 前端 (網頁介面) | React + Vite | **Vercel** |
| 後端 (API 伺服器) | Node.js + Express | **Render** |
| 資料庫 | Turso (LibSQL 雲端) | **Turso Cloud** |
| 圖片託管 | Cloudinary | **Cloudinary CDN** |
| 登入驗證 | Firebase Authentication（Google 登入）+ 共用幹部名單 | **Firebase** |

> 💡 **重點**：網站資料存在 Turso 雲端資料庫，只要登入後台就能隨時隨地修改內容，**不需要動到任何程式碼**。

## 管理員登入

網站後台改用 **Google 登入**（與 ITRC 幹部工具站同一套帳號系統），不再有共用帳號密碼。身分是用你登入的 Google email 去比對社團共用的**幹部名單**判定的。

### 步驟

1. 打開瀏覽器，前往網站網址後面加上 `/admin`（或 `/login`），例如：`https://nsysu-itrc.vercel.app/admin`
2. 點「**使用 Google 登入**」，在跳出的 Google 視窗選你的帳號登入
3. 系統會拿你的 email 比對幹部名單：只要在名單上（角色不是「無權限」），就會進入管理後台 (`/admin`)
4. 若被擋下，代表你的 email 不在名單上 —— 請找社長或顧問處理（見下方 FAQ）

> ⚠️ **重點**：
> - 幹部名單維護在 **ITRC 幹部工具站的「幹部與權限」頁**（與工具站共用同一份雲端名單）。要讓某人能登入官網後台，就在那頁把他的 Google 信箱加進名單；**本人用那個 Google 帳號**登入才生效，信箱留空的人無法登入。
> - 目前設計為：**只要在名單上的人，登入後都有完整的後台編輯權限**（尚未細分誰能改哪一塊）。所以只把真正需要編輯網站的人加進名單。
> - 內建兩個**萬能管理員**帳號（`zhongshantouyan@gmail.com`、`hades60414@gmail.com`），不受名單影響、永遠可登入，避免全屆幹部離開後沒人能進的死鎖。

## 1. 頁面內容管理

> 用於修改首頁上顯示的文字內容

### 可修改的區塊

| 區塊 Key | 說明 |
|----------|------|
| `hero_title` | 首頁大標題（例如：中山投研） |
| `hero_subtitle` | 首頁副標題 |
| `course_intro` | 課程簡介段落 |
| `course_info` | 課程資訊（時間、地點、聯絡方式等） |

### 操作步驟

1. 點選「**頁面內容**」分頁
2. 找到你想修改的區塊，點擊右側的「**編輯**」按鈕
3. 修改「**標題**」和「**內容**」欄位
4. 點擊「**儲存**」
5. 看到 ✓ 已儲存 的提示後，回到前台重整頁面即可看到變更

> 📝 **注意**：`course_info` 區塊的內容是 JSON 格式，修改時請保持格式正確：
> ```json
> {
>   "time": "每週五 18:00 - 21:00",
>   "location": "管理學院 CM1037",
>   "coordinators": "張大恒、李沁彤、許筠采",
>   "email": "zhongshantouyan@gmail.com",
>   "ig": "nsysu_itrc"
> }
> ```

## 2. 成果發表管理

> 用於管理社團的比賽成績、研究成果等展示

### 新增成果

1. 點選「**成果發表**」分頁
2. 在上方表單填寫：
   - **學期**（必填）：例如 `114-1`
   - **類別**（選填）：例如 `台股`、`美股`、`加密貨幣`
   - **標題**（必填）：成果名稱
   - **說明**（選填）：簡要描述
   - **連結**（選填）：相關資料連結
3. 點擊「**新增**」

### 編輯 / 刪除成果

- 在列表中找到目標項目
- 點「**編輯**」→ 修改表單內容 → 點「**更新**」
- 點「**刪除**」→ 確認對話框中點「確定」即可刪除

## 3. 社團成員管理

> 用於管理幹部與社員名單

### 新增成員

1. 點選「**社團成員**」分頁
2. 填寫：
   - **姓名**（必填）
   - **職位**（選填）：例如 `社長`、`副社長`、`公關長`、`社員`
   - **系所**（選填）：例如 `財管系`
   - **年級**（選填）：例如 `大三`
   - **大頭照**（選填）：可直接上傳圖片或貼上圖片網址
3. 點擊「**新增成員**」

### 編輯 / 刪除成員

- 在列表中點「**編輯**」可修改資訊（列表會顯示大頭照縮圖）
- 點「**刪除**」可移除成員

> 💡 **學期交接提醒**：每學期交接時，記得更新幹部名單！

## 4. 活動管理

> 用於管理活動規劃（未來活動）與活動紀錄（已完成活動）

### 新增活動

1. 點選「**活動管理**」分頁
2. 填寫：
   - **類型**（必填）：下拉選擇「活動規劃」或「活動紀錄」
   - **日期**（選填）：例如 `114年09月26日(五)`
   - **主題**（必填）：活動名稱
   - **主講人**（選填）：例如 `王昭文 教授`
   - **備註/其他連結**（選填）：支援 Markdown 語法
   - **活動照片**（選填）：見下方上傳說明
3. 點擊「**新增活動**」

### 上傳圖片（活動照片 / 成員大頭照）

所有圖片上傳都使用 **Cloudinary** 雲端託管，有兩種方式：

**方式一：直接上傳圖片**
1. 點擊「選擇檔案」按鈕
2. 選擇電腦中的圖片（上限 10MB）
3. 系統會自動上傳到 Cloudinary 雲端並填入圖片網址
4. 看到 ✓ 圖片上傳成功 即代表完成
5. Cloudinary 會自動幫你壓縮、優化圖片格式，網頁載入更快！

**方式二：貼上圖片網址**
- 直接在圖片網址欄位貼上任何公開圖片的 URL

> ⚠️ 上傳圖片需要設定 Cloudinary API 金鑰，詳見 [外部服務帳號資訊](#外部服務帳號資訊)

### 活動類型說明

| 類型 | 說明 | 對應前台頁面 |
|------|------|-------------|
| `活動規劃 (plan)` | 未來預計舉辦的活動 | `/plans` 活動規劃頁 |
| `活動紀錄 (record)` | 已完成的活動記錄 | `/activities` 活動紀錄頁 |

## 修改版面設計 / 程式碼

> ⚡ 以下流程只適用於需要修改**網頁設計、排版或新增功能**的情況
>
> 如果你只是要更新文字、成員、活動等內容，**請使用上面的後台管理**，不需要看這一段。

### 前置準備

確保你的電腦已安裝：
- [Node.js](https://nodejs.org/)（建議 v18+）
- [Git](https://git-scm.com/)
- [VSCode](https://code.visualstudio.com/)（推薦）

### 修改 → 部署流程

```
修改程式碼 → git add . → git commit → git push → 自動部署 ✨
```

#### 詳細步驟：

1. **在 VSCode 中修改檔案**
   - 改顏色/樣式 → 編輯 `client/src/index.css`
   - 改頁面排版 → 編輯 `client/src/pages/*.jsx` 或 `client/src/components/*.jsx`
   - 改後端邏輯 → 編輯 `server/routes/*.js`

2. **存檔後，打開終端機執行 Git 指令**

   ```bash
   git add .
   git commit -m "簡短描述你改了什麼，例如：更新首頁顏色"
   git push
   ```

3. **等待自動部署**
   - **Vercel**（前端）和 **Render**（後端）會自動偵測 GitHub 上的新程式碼
   - 約 **1~2 分鐘**後重新整理網站即可看到更新

> 💡 正常情況下不需要手動去 Vercel/Render 按 Redeploy。只有在 bug 或首次設定失敗時，才需要到後台手動操作。

## 常見問題 FAQ

### Q: 登入不進去 / 顯示沒有權限怎麼辦？

現在是 Google 登入、沒有密碼。進不去通常兩種情況：(1) 你登入的 Google email 不在幹部名單上 —— 請社長或顧問到工具站「幹部與權限」頁把你的信箱加進去；(2) 你登錯 Google 帳號 —— 登出換成名單上的那個帳號。若全屆幹部都離開、沒人能編名單，用內建的**萬能管理員**帳號（見「管理員登入」）登入解鎖。

### Q: 修改內容後，前台沒有更新？

1. **硬重整**瀏覽器：按 `Cmd + Shift + R`（Mac）或 `Ctrl + Shift + R`（Windows）
2. 確認儲存時有看到 ✓ 已儲存 的提示
3. 確認後端服務 (Render) 是否正常運作

### Q: Render 後端好像休眠了？

Render 免費方案的伺服器閒置約 15 分鐘後會自動休眠，第一次存取時需等待約 30 秒～1 分鐘喚醒。可使用 [cron-job.org](https://console.cron-job.org/jobs) 設定定時 ping 來保持伺服器活躍。

### Q: 圖片上傳失敗？

1. 確認 Render 環境變數中有設定 `CLOUDINARY_CLOUD_NAME`、`CLOUDINARY_API_KEY`、`CLOUDINARY_API_SECRET`
2. 確認圖片大小未超過 10MB
3. 確認網路連線正常
4. 確認 Cloudinary 帳號免費額度尚未用完

### Q: 我可以同時有多個管理員嗎？

可以。現在是多人各自用自己的 Google 帳號登入，只要 email 在幹部名單上即可，不需要共用帳密。要新增管理員，就在工具站「幹部與權限」頁加他的 Google 信箱。

## 外部服務帳號資訊

### Cloudinary（圖片上傳用）⭐ 重要

1. 前往 [Cloudinary 官網](https://cloudinary.com/) 註冊免費帳號
2. 登入後，在 Dashboard 首頁就能看到你的 **Cloud Name**、**API Key**、**API Secret**
3. 到 **Render 專案後台** → Environment → Environment Variables
4. 新增以下 3 個環境變數：

   | Key | Value |
   |-----|-------|
   | `CLOUDINARY_CLOUD_NAME` | Dashboard 上的 Cloud Name |
   | `CLOUDINARY_API_KEY` | Dashboard 上的 API Key |
   | `CLOUDINARY_API_SECRET` | Dashboard 上的 API Secret |

5. 儲存後，手動點 **Manual Deploy** 讓設定生效

> 💡 **Cloudinary 免費方案**：每月 25 credits（約 25GB 儲存空間或 25,000 次轉換），小型社團網站綽綽有餘！
> 上傳的圖片會自動壓縮、轉換格式（WebP/AVIF），並透過全球 CDN 加速載入。

### Cron Job（防止後端休眠）

- 管理頁面：[https://console.cron-job.org/jobs](https://console.cron-job.org/jobs)
- 設定每 14 分鐘 ping 一次你的後端 health check endpoint：
  ```
  https://你的後端網址.onrender.com/api/health
  ```
